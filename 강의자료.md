근데 우리가 토이 프로젝트를 만들기에 앞서 리액트가 뭔지부터 배워야겠죠
좀 지루하겠지만 그래도 프로젝트를 만들기 전에 꼭 알아야 하는 내용이니 꼭 들어주길 바랄게요..
우리가 이때까지 학교에서 웹페이지를 만들때 뭘 사용했음?
Html, css, js를 사용했을거임.
Html, css, js 를 사용하여 웹개발을 할때에는 어떻게 했죠?
.html파일 하나에 모든 내용을 전부 작성하거나, 각각 .js, .css, .html파일을 작성했죠?
하지만 현대의 웹은 이 방식으로 작성하기에는 너무나 많은 길을 와버렸습니다…

초기 웹은 진짜 말 그대로 정보를 표시하는, 글 위주의 정적인 웹사이트들이 많았어요.
하지만 점점 html, css, js 에 여러 기능들이 추가되고, 사용자들이 상호작용할 수 있는 기능들이 추가되면서
(웹사이트 예시)
이런 사이트만 보더라도 하나의 페이지에 이렇게 많은 요소들이 있는데, 하나의 html파일로 작성하기에는 많이 무리가 있는건 사실이죠.
게다가 js코드를 작성해서 구현해줄 기능들도 한두개가 아니잖음.

리액트 공식 문서의 소개문 첫 줄에도
“React는 사용자 인터페이스를 구축하기 위한 선언적이고 효율적이며 유연한 JavaScript 라이브러리입니다.”
라고 쓰여있죠. 일단 지금은 “아~ JS를 기반으로 웹페이지를 더 쉽고 편하게 만들 수 있게 해주는 라이브러리구나”
라고 생각하면 됩니다.

그래서 리액트가 사용자 인터페이스를 구축하기 위해 선언적이고 효율적이며 유연하게 동작할 수 있는 이유는 무엇일까요?
이는 크게 리액트의 동작 방식, 문법에 있습니다.

먼저 동작 방식부터 살펴봅시다.

이에 관한 공식문서의 표현을 인용하면,
“React로 사용자 인터페이스를 빌드할 때, 먼저 이를 컴포넌트라는 조각으로 나눕니다. 그리고 각 컴포넌트의 다양한 시각적 상태들을 정의합니다. 마지막으로 컴포넌트들을 연결하여 데이터가 그 사이를 흘러가게 합니다.”

라고 하죠. 지금은 이게 무슨 개소리냐 할 수 있습니다.
(유튜브 페이지 열기)
유튜브 많이 보시죠.
이 유튜브의 메인 페이지를 어떻게 그려야 효율적일까요. 메인 페이지를 자세히 보시면, 반복되는 레이아웃들이 보이지 않나요?
아까 공식문서의 표현에 ‘React로 사용자 인터페이스를 빌드할 때, 먼저 이를 컴포넌트라는 조각으로 나눕니다.’ 라는 표현,
잊지 않으셨죠. 저렇게 반복되는 요소들을 ‘컴포넌트’ 라는 하나의 단위로 묶어 UI를 구성할 때 효율적으로 활용할 수 있게 되는거임.

그리고 문법인데요, 리액트에서는 jsx라는 문법을 사용합니다.
jsx가 무엇이냐, javascript + xml, js내부에서 html을 사용할 수 있게 하겠다라는 의미입니다.
컴포넌트를 담고 있는 파일을 이 jsx확장자로 생성하게 됩니다.
따라서 기본 구조는 헷갈려할 필요 없이 컴포넌트 파일은 jsx, 다른 일반 파일은 js로 생성하는구나 라고 알고 계시면 됩니다.

오늘 강의를 끝까지 다 따라오신다면..
웬만한 멋진 사이트 하나정도는 혼자 만들 수 있을겁니다!
물론 따라오지 못해도 괜찮음. 노베이스로 이걸 따라온다? 괴물입니다 그냥
저도 괴물은 아니였어서 저였어도 이거 따라오긴힘들것.
원래 개발은 남의코드 보면서 배우는게 제일 쉬워요. 이해가안되면 코드유심히보면서 찾아보고 ai한테 로직분석 시키면서 본인껄로 만들어 나갑시다.
그렇다고 제 코드가 정답은 아님. 계속 코드와 아키텍처에 대해 고민하는 시간이 프론트엔드 개발자에게는 필요함.
저도 엄청 많이 고민했었고, 아직도 정답을 찾지는 못했음.
